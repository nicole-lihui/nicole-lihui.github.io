# 微服务

## 架构演进
* 传统的单体架构
  * 当访问量加大时，达到一定数量级，无法负载。
* RPC垂直分层架构（dubbo）
  * 支持高并发
  * 耦合大，解耦困难
  * 语言支持弱（一般单语言）
  * 链路查错弱
  * 单体速度快
* 微服务
  * 单体速度相对rpc慢
  * 绝大部分都是基于restful（便于扩展，与维护）
  * 支持高并发
  * 微服务过多，服务治理成本高，不利于系统维护
  * 使用容器，更容易部署
* 服务网格 service mesh
  * 削弱的java的依赖性
* 中台
* 无服务与零原生
